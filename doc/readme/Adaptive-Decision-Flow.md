## Adaptive Decision Flow

The adaptive decision flow is designed to maximize student agency while maintaining pedagogical rigor. When a concept is selected—either through the Progression Fringe of the Knowledge Structure Theory (KST) subsystem or the Review Queue generated by the Retention/Decay subsystem—the platform activates an Adaptive Recommendation Engine and presents a Learning Fork.

This mechanism combines system guidance with student autonomy, allowing the learner to either study the material or directly challenge the concept through an assessment.

---

### Phase 1: Recommendation Engine
Before presenting the learning fork, the system evaluates the learner’s current knowledge state using signals from the Concept Mastery Subsystem (BKT) and Retention Subsystem (Decay Model).

The engine estimates the learner’s effective mastery level and provides a recommended pathway.

**Decision logic:**
*(System evaluates mastery thresholds to recommend either instructional review or direct assessment)*

The recommendation is advisory; the student retains final control over the pathway selection. This design supports self-regulated learning while still leveraging adaptive intelligence.

---

### Phase 2: The Learning Fork
Once a concept is targeted, the system presents two learning pathways:
* **Instructional Path (Learn):** For students who are new to the concept or require guided learning.
* **Assessment Path (Challenge / Test-Out):** For students who believe they already possess sufficient knowledge and wish to skip instruction.

---

### Phase 3: Pathway Logic

#### Pathway A: Instructional Path (Learn)
* **Step 1: Course Content**
  The student first accesses the static instructional material derived from the Grade 12 Mathematics curriculum. This includes:
  * Concept explanations
  * LaTeX-rendered formulas
  * Worked examples
  * Visual illustrations
  
  *This ensures curriculum integrity and consistency across learners.*

* **Step 2: Adaptive Practice**
  While completing the lesson, the student enters a low-stakes adaptive practice environment. Features include:
  * Questions served according to difficulty tiers
  * Question selection guided by the student’s estimated ability
  * AI Socratic Tutor providing structured hints
  * Immediate feedback for incorrect answers
  
  *Performance during this stage is used to prepare the learner for the mastery assessment, but does not update the permanent mastery estimate in the BKT model.*

* **Step 3: Checkpoint Layer**
  Before accessing the mastery exam, the learner must pass a Checkpoint Question. Checkpoint questions serve as a competency gate, ensuring that the learner demonstrates minimal understanding before attempting the formal assessment.
  * **Checkpoint logic:**
    * **Correct answer** $\\rightarrow$ Unlock Mastery Exam
    * **Incorrect answer** $\\rightarrow$ Assign additional adaptive practice questions
  
  *This mechanism prevents students from advancing through guessing or superficial practice.*

* **Step 4: Mastery Exam**
  Once the checkpoint is passed, the student may attempt the Concept Mastery Exam.

#### Pathway B: Assessment Path (Challenge)
Students selecting the Challenge Path bypass instructional content and practice.

* **Step 1: Direct Concept Exam**
  The learner immediately attempts a timed concept exam modeled after the Ethiopian Higher School Leaving Certificate Examination (EHSLCE).

* **Step 2: Non-Assisted Assessment Environment**
  To ensure diagnostic validity:
  * AI hints are disabled
  * No guided feedback is provided
  * A timer is enforced
  
  *This assessment functions as a diagnostic measurement of existing knowledge.*

---

### Phase 4: Exam Attempt Policy
To maintain the integrity of the assessment process, the platform enforces an Exam Attempt Policy.

**Rules include:**
* Maximum number of attempts per concept exam
* Mandatory practice completion after repeated failures
* Optional cooldown period between attempts

**Example policy:** * **Maximum attempts:** 3
* If the learner fails the exam twice, the system requires additional practice questions before another attempt.

This prevents exam spamming and improves the reliability of mastery estimation.

---

### Phase 5: Universal Mastery Update (BKT Engine)
Regardless of the pathway selected, the Concept Exam acts as the authoritative evaluation stage. During this stage, the system records:
* Question-level responses
* Total exam score
* Time spent per question

These results are passed to the Bayesian Knowledge Tracing (BKT) Engine.

**Mastery update logic:**
* **If the learner passes the exam:** $P(L)$ is set to a high mastery probability (e.g., $0.90 – 0.95$).
* **If the learner fails:** $P(L)$ remains low and the system redirects the learner to the Instructional Path for guided learning.

---

### Phase 6: Knowledge Graph Update (KST Engine)
After the BKT update, the Knowledge Structure Subsystem evaluates prerequisite relationships within the Directed Acyclic Graph (DAG) of concepts.

**If the mastery threshold is reached:**
The system unlocks dependent concepts and updates the student’s Progression Fringe. This enables dynamic curriculum progression tailored to the learner’s mastery state.

---

### Phase 7: Learning Analytics Capture
Throughout the entire decision flow, the system records learning activity through the Learning Analytics Subsystem.

**Collected metrics include:**
* Number of practice attempts per concept
* Time spent studying instructional material
* Exam attempts and scores
* Checkpoint success rate
* Concept mastery progression

**These analytics support:**
* Adaptive recommendations
* Instructor dashboards
* Future improvements to the adaptive algorithms

---

### Phase 8: State Transition Summary
The learner’s state for any given concept evolves according to the following finite-state model:

| Current State | Action | Next State |
| :--- | :--- | :--- |
| **Locked** | Pre-requisites met | **Unlocked (Fringe)** |
| **Unlocked** | Student selects Learn | **In-Progress (Practice)** |
| **Unlocked** | Student selects Challenge | **Challenging** |
| **Challenging** | Fail Exam | **In-Progress (Redirect to Learn)** |
| **In-Progress** | Pass Mastery Exam | **Mastered** |
| **Mastered** | Time elapses + knowledge decay | **Review Needed** |
